## Памятка
Для удобства работы над проектом используются инструменты из **Node.js** и **npm**.

Клонируйте репозиторий на свой компьютер
<img width="769" alt="" src="https://user-images.githubusercontent.com/10909/35275224-1d97a7dc-0050-11e8-89f9-41c9a7b2ae9c.png">

Клонируйте репозиторий на свой компьютер:

```
git clone https://github.com/KristinaVavilina/Kekstagram.git
```

Перейдите в папку проекта:

```
cd Kekstagram
```

Установите зависимости:

```
npm install
```

Запустите проект:

```
npm start
```

## Структура проекта

### `css/`, `fonts/`, `img/`

Директории со статическими файлами проекта: стилями, изображениями, шрифтами и т.д.

### `photos/`

Директория с фотографиями, которые нужно будет отобразить на странице в одном из заданий.

### `js/`

Ваша главная рабочая директория, в которой будут храниться все скрипты проекта.

### `index.html`

Главная страница проекта.

### Остальное

Все остальные файлы в проекте являются служебными.

## Используемые технологии

### Фронтенд

* **HTML5**
* **CSS3**
* **JavaScript (Vanilla JS)**: Чистый JavaScript используется для всей клиентской логики, включая:
    * Взаимодействие с API (`api.js`).
    * Обработка и валидация форм (`submit-form.js`).
    * Применение графических фильтров и эффектов (`filters.js`, `switch-effects.js`).

### Среда разработки и инструменты

* **Node.js / npm (или Yarn)**: Используется для управления зависимостями проекта (наличие `package.json`).
* **ESLint**: Применяется для статического анализа кода, обеспечения его качества и единообразного стиля.
* **EditorConfig**: Для унификации настроек форматирования кода среди разных разработчиков и редакторов.
* **Git / GitHub**: Система контроля версий и платформа для хостинга проекта.

## Основные возможности

1. Загрузка нового изображения на сайт и заполнение информации о нём
2. Редактирование изображения и ограничения, накладываемые на поля
3. Отправка данных на сервер
4. Просмотр загруженных изображений
5. Фильтрация изображений от других пользователей
